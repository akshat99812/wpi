## ⚠️ Headline finding — the runbook models a DIFFERENT engine

The runbook (v2) assumes a **cell-based H3 suitability mapper**: H3 tiling → per-cell score →
heatmap → top-cells / "best-pocket" ranking → 5 classes incl. **POOR** → exclusion buffers
that gate scoring → UTM reprojection → bilinear resampling → multi-cell aggregation.

The **actual** deployed engine (`apps/api/src/services/analysis/`) is a **single-AOI,
whole-polygon scalar screener**: one validated polygon → one stitched GWA raster patch per
layer at a fixed zoom (z10) → one boolean in/out **pixel** mask → AOI-wide **scalar** stats →
one 0–100 score (4 weighted components) → one flat `AnalysisResponse` (score + 5 degradable
sections). **~9 of the runbook's 14 assumptions are ABSENT or DIFFERENT.**

Full evidence: [`RECONCILIATION_AND_DEFERRED.md`](./RECONCILIATION_AND_DEFERRED.md) (76 mismatches).
